## 6. Бизнес-модель

*   **Free Plan:** Бесплатное хранение и обмен зашифрованными сниппетами.
*   **Premium Plan:** Ежемесячная подписка, включающая пакет кредитов для использования премиум-функций.
*   **Pay-as-you-go:** Возможность пополнять баланс (кредиты) напрямую через платежный шлюз (ЮKassa, Stripe).
*   **Стоимость операций (примерная):**
    *   Запуск кода в Docker: `1 кредит`.
    *   Аудит кода через ИИ: `15 кредитов`.

### Гостевой доступ (Основной поток)

**Цель:** Минимальный барьер входа для быстрого обмена кодом.

**Поток пользователя:**
1. Пользователь заходит на `codetesting.ru`
2. Сразу видит форму для создания сниппета
3. Вводит код, выбирает язык и тему
4. Нажимает "Создать"
5. Получает:
   - **Ссылку на сниппет** (например: `codetesting.ru/c/abc123`)
   - **Токен для редактирования** (например: `tk_xyz789`)

- ✅ Нет регистрации
- ✅ Мгновенный доступ

### Fingerprint система

**Цель:** Улучшение UX и дополнительная безопасность через автоматическое распознавание пользователей.

**Как это работает:**
1. **Создание fingerprint:**
   - Браузер отправляет данные: User-Agent, разрешение экрана, часовой пояс, языки
   - Система генерирует уникальный fingerprint (например: `fp_abc123xyz`)
   - Сохраняется в localStorage и отправляется на сервер

2. **Автоматическое связывание:**
   - При создании сниппета система проверяет fingerprint
   - Если найден существующий пользователь с таким fingerprint → предлагает связать сниппеты
   - Если новый fingerprint → создает новую "сессию"

3. **Улучшенный UX:**
   - Пользователь видит: "Найдено 3 ваших сниппета. Показать?"
   - Возможность связать fingerprint с аккаунтом
   - Автоматическое восстановление доступа при очистке cookies

**Техническая реализация:**

**Таблица `fingerprints`:**
```sql
CREATE TABLE fingerprints (
    id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    fingerprint_hash VARCHAR(64) NOT NULL UNIQUE,
    user_id BIGINT UNSIGNED NULL, -- NULL для гостей
    browser_data JSON, -- User-Agent, разрешение, часовой пояс
    created_at TIMESTAMP NULL,
    last_seen_at TIMESTAMP NULL,
    
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE SET NULL
);
```

**Таблица `codes` дополняется:**
```sql
CREATE TABLE codes (
    -- ... существующие поля ...
    fingerprint_id BIGINT UNSIGNED NULL,
    -- ... остальные поля ...
    
    FOREIGN KEY (fingerprint_id) REFERENCES fingerprints(id) ON DELETE SET NULL
);
```

**JavaScript для создания fingerprint:**
```javascript
// Создание fingerprint на клиенте
const createFingerprint = () => {
    const data = {
        userAgent: navigator.userAgent,
        screen: `${screen.width}x${screen.height}`,
        timezone: Intl.DateTimeFormat().resolvedOptions().timeZone,
        languages: navigator.languages.join(','),
        platform: navigator.platform,
        cookieEnabled: navigator.cookieEnabled,
        doNotTrack: navigator.doNotTrack
    };
    
    // Хеширование данных
    return btoa(JSON.stringify(data)).substring(0, 32);
};
```

**Безопасность и ограничения:**
- ✅ **Анонимность:** Fingerprint не содержит личных данных
- ✅ **Rate limiting:** Ограничения по fingerprint
- ✅ **Мониторинг:** Отслеживание подозрительной активности
- ✅ **Право на забвение:** Возможность удалить fingerprint
- ⚠️ **Не идеально:** Fingerprint может меняться при смене браузера/устройства

**UX особенности:**
- Уведомление: "Мы используем fingerprint для вашего удобства. Это помогает восстановить доступ к сниппетам."
- Кнопка "Не показывать мои сниппеты" для приватности
- Возможность отключить fingerprint в настройках
- Автоматическое предложение регистрации при частом использовании

### 9.3. Архитектурные решения для проекта

#### Backend архитектура
- **Domain-Driven Design (DDD):** Организация кода по доменам (User, Code, Execution)
- **Repository Pattern:** Абстракция доступа к данным
- **Service Layer:** Бизнес-логика в сервисах
- **Event Sourcing:** Логирование всех изменений сниппетов
- **CQRS:** Разделение команд и запросов

#### Frontend архитектура
- **Component Architecture:** Модульная архитектура компонентов
- **Composition API:** Современный подход к логике компонентов
- **Pinia Store:** Централизованное управление состоянием
- **Composables:** Переиспользуемая логика

#### Безопасность
- **Defense in Depth:** Многоуровневая защита
- **Principle of Least Privilege:** Минимальные привилегии
- **Zero Trust:** Не доверяй, проверяй

#### Производительность
- **Lazy Loading:** Загрузка по требованию
- **Caching Strategy:** Стратегии кэширования
- **Database Optimization:** Оптимизация запросов
- **CDN:** Доставка статического контента

#### Тестирование стратегия
- **Pyramid Testing:** Больше unit тестов, меньше E2E
- **Test-Driven Development:** Разработка через тестирование
- **Behavior-Driven Development:** Поведенческое тестирование

#### Мониторинг и логирование
- **Structured Logging:** Структурированное логирование
- **Application Performance Monitoring:** Мониторинг производительности
- **Error Tracking:** Отслеживание ошибок
- **User Analytics:** Аналитика пользователей
